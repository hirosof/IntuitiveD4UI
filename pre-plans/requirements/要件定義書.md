## 1. プロジェクト概要

### 1.1 目的

Webサイト/Webアプリケーションおよびデスクトップアプリケーションのワイヤーフレームを作成するためのツールを開発する。

既存のワイヤーフレーム作成ツールでは、HTML の DOM 構造のような要素の親子関係（入れ子構造）を自然に表現することが困難であり、例えば `<div class="parent"><div class="child"></div></div>` のような構造を表すには複数の図形を手動で重ねる必要がある。本ツールではこの課題を解決し、要素のコンテナ構造をネイティブにサポートする。

### 1.2 対象ユーザー

- 開発者本人のみ（個人利用）

### 1.3 ワイヤーフレーム作成対象

- Webサイト / Webアプリケーション
- デスクトップアプリケーション（Windows）

---

## 2. ツール形態

### 2.1 アプリケーション形態

- **検討中**: Webアプリケーション
- 最終決定は基本設計フェーズで行う

### 2.2 動作要件

- **オフライン動作**: ネットワーク接続がない環境でも、ツールの基本機能が使用可能であること
- オンライン時に追加機能（データ同期等）が利用可能となる構成も許容するが、オフラインでの基本動作は必須要件とする

※ 具体的な動作モード（オンライン/オフラインの切り替え方式等）はアプリケーション形態の決定後に基本設計で策定する。

## 3. 機能要件

### 3.1 機能分類一覧

本ツールの機能は以下の7つの分類で構成される。

| No. | 分類名 | 概要 | 詳細セクション |
|-----|--------|------|:--------------:|
| 1 | キャンバス操作 | ワイヤーフレーム作業領域の操作 | 3.2 |
| 2 | 基本図形要素 | 基本的な図形の描画・配置 | 3.3 |
| 3 | UIコンポーネント | プリセットUIパーツの配置 | 3.4 |
| 4 | 要素構造・コンテナ管理 | 要素の親子関係と構造管理 | 3.5 |
| 5 | レイアウト・編集 | 要素の配置調整と編集操作 | 3.6 |
| 6 | プロジェクト管理 | プロジェクト・ページの管理 | 3.7 |
| 7 | エクスポート | 各種形式への出力 | 3.8 |

### 3.2 キャンバス操作

ワイヤーフレームを作成する作業領域に関する機能。

#### 3.2.1 キャンバスサイズ

- **プリセットサイズ**: 一般的な画面サイズのプリセットを用意する
  - 例: デスクトップ（1920x1080, 1366x768 等）、タブレット（1024x768, 768x1024 等）、モバイル（375x667, 414x896 等）
  - プリセットの具体的な一覧は基本設計で確定する
- **自由指定**: 任意の幅・高さをピクセル単位で指定可能

#### 3.2.2 選択操作

キャンバス上の要素に対して、以下の選択操作が可能であること。

- **クリック選択**: 要素をクリックして単一選択
- **範囲選択**: ドラッグによる矩形範囲内の要素を一括選択
- **追加選択**: Shift+クリックで選択に追加/除外（トグル）
- **全選択**: すべての要素を一括選択（キーボードショートカット対応）

#### 3.2.3 ドラッグ＆ドロップ

- 要素をキャンバス上にドラッグ＆ドロップで配置できること
- 配置済み要素をドラッグで移動できること

#### 3.2.4 ズーム・パン

- キャンバスのズームイン・ズームアウトが可能であること
- キャンバスのパン（スクロール・移動）が可能であること

#### 3.2.5 グリッド・スナップ

- グリッド表示のON/OFFが切り替えられること
- グリッドへのスナップ（吸着）のON/OFFが切り替えられること
- グリッドの間隔を設定可能であること

#### 3.2.6 ガイドライン

- 要素の整列を補助するガイドラインを表示できること
- 要素移動時に他要素との位置関係をスマートガイドで表示すること

### 3.3 基本図形要素

ワイヤーフレーム上に配置可能な基本的な図形要素。

#### 3.3.1 対応図形

| 図形 | 説明 |
|------|------|
| 矩形 | 角丸対応。幅・高さ・角丸半径を指定可能 |
| 円/楕円 | 幅・高さで楕円を表現 |
| 線 | 直線。始点・終点を指定 |
| 矢印 | 方向を示す矢印。始点・終点、矢印の向きを指定 |
| テキスト | 文字列の配置。フォントサイズ、太字/斜体等の基本的な書式設定 |
| 画像プレースホルダー | 画像の配置位置を示すプレースホルダー。サイズ指定可能。※実画像のインポート・配置は対象外とし、あくまでプレースホルダー（位置・サイズの占有表示）のみとする |

#### 3.3.2 共通プロパティ

すべての図形要素は以下の共通プロパティを持つ。

- 位置（X, Y座標）
- サイズ（幅、高さ）
- 回転角度
- 不透明度
- 表示/非表示
- ロック（編集不可状態）

### 3.4 UIコンポーネント

よく使われるUI要素をプリセットとして用意する。

#### 3.4.1 対応コンポーネント

| コンポーネント | 説明 |
|----------------|------|
| ボタン | テキスト付きボタン |
| 入力フィールド | テキスト入力欄。プレースホルダーテキスト対応 |
| ドロップダウン | 選択肢リストを持つドロップダウン |
| チェックボックス | ラベル付きチェックボックス |
| ラジオボタン | ラベル付きラジオボタン |
| ナビゲーションバー | 上部ナビゲーションバー |
| サイドバー | 左右のサイドバー |
| リスト | 項目の一覧表示 |
| テーブル | 行・列のテーブル表示 |
| カード | コンテンツをカード形式で表示 |
| モーダル/ダイアログ | オーバーレイ表示されるダイアログ |

#### 3.4.2 コンポーネントのカスタマイズ

- 各コンポーネントのサイズ変更が可能であること
- テキスト内容の変更が可能であること
- 基本的なスタイル（色、枠線等）の変更が可能であること

### 3.5 要素構造・コンテナ管理

**本ツールの中核的な特徴機能**。
HTML の DOM 構造のような要素の親子関係（入れ子構造）をネイティブにサポートする。

#### 3.5.1 親子関係の設定

要素間の親子関係を以下の2つの方法で設定できること。

- **ドラッグ＆ドロップによる自動認識**: 要素を別の要素（コンテナ）上にドロップすると、自動的に親子関係が成立する
- **明示的な操作**: メニューやショートカット等から「子要素として追加」を明示的に実行できる

#### 3.5.2 コンテナの動作

- 親要素を移動すると、子要素も追従して移動すること
- 親要素のリサイズ時、子要素が適切に振る舞うこと
- 親子関係の深さ（ネスト）に制限を設けないこと（実用上の制限は基本設計で検討）

#### 3.5.3 スペーシング（余白）

以下のスペーシング（余白）プロパティをサポートする。

- **パディング（内側余白）**: コンテナ要素が持つプロパティ。コンテナの内側と子要素の間の余白。上下左右を個別に指定可能
- **マージン（外側余白）**: コンテナ要素およびコンテナの子要素が持つプロパティ。要素の外側の余白。上下左右を個別に指定可能

※ パディングはコンテナ要素専用。マージンはコンテナ要素自身と、コンテナ内に配置された子要素の両方に適用される。コンテナに属さないトップレベルの基本図形にはマージンは適用されない。

#### 3.5.4 自動レイアウト

コンテナ内の子要素に対して、以下の自動レイアウト機能を提供する。

- **自由配置**: 子要素を親要素内の任意の位置に配置（絶対位置指定）
- **自動レイアウト（Flexbox的）**: 子要素を水平方向または垂直方向に自動配置
  - 配置方向（横並び/縦並び）
  - 要素間のギャップ（間隔）
  - 折り返しの有無
  - 主軸・交差軸の配置（左寄せ、中央、右寄せ、均等配置等）
  - 子要素のサイズ挙動: 固定サイズの維持、または親コンテナに応じた伸縮（flex-grow的な挙動）を要素ごとに指定可能
- **自動レイアウト（Grid的）**: 子要素をグリッド状に自動配置
  - 列数・行数の指定
  - セルギャップの指定
  - セル内の子要素のサイズ挙動: 固定サイズの維持、またはセルサイズに応じた伸縮を要素ごとに指定可能

#### 3.5.5 ツリービュー（構造パネル）

- キャンバス上の全要素を、親子関係を反映したツリー構造で表示するパネルを提供すること
- ツリービュー上での操作:
  - 要素の選択（キャンバス上の該当要素がハイライト）
  - ドラッグ＆ドロップによる親子関係の変更
  - 要素の表示/非表示の切り替え
  - 要素名の変更

### 3.6 レイアウト・編集

要素の配置調整と一般的な編集操作。

#### 3.6.1 整列・均等配置

- 複数要素の整列（左揃え、中央揃え、右揃え、上揃え、中央揃え、下揃え）
- 複数要素の均等配置（水平方向、垂直方向）

#### 3.6.2 グループ化

- 複数要素をグループとしてまとめることができること
- グループの解除が可能であること
- グループ内の個別要素を直接編集可能であること（ダブルクリック等で）

※ グループ化は「要素構造・コンテナ管理」（3.5）の親子関係とは異なる概念。

| 観点 | グループ | コンテナ（親子関係） |
|------|----------|---------------------|
| 目的 | 表示上の便宜のためのまとまり | 構造的な入れ子関係の表現 |
| レイアウト | 子要素の位置関係は固定 | 自動レイアウト（Flexbox/Grid的）が適用可能 |
| パディング/マージン | なし | あり |
| ツリービューでの表現 | フラットな要素のグループ表示 | 親子階層として表示 |

以下のエッジケースについて、具体的な動作は基本設計で定義する。
- グループ内の要素をコンテナにドロップした場合の動作
- グループをコンテナに変換する機能の要否
- コンテナ内の要素をグループ化した場合の親子関係の扱い

#### 3.6.3 レイヤー管理

- 要素の前面/背面の順序（Zオーダー）を変更できること
  - 最前面へ移動
  - 1つ前面へ移動
  - 1つ背面へ移動
  - 最背面へ移動

#### 3.6.4 コピー＆ペースト

- 要素のコピー＆ペーストが可能であること
- 要素の複製（その場でコピーを作成）が可能であること
- 要素のカット＆ペーストが可能であること
- **ページ間の要素コピー＆ペースト**が可能であること（異なるページへの要素の移動・複製）

#### 3.6.5 Undo/Redo

- 操作の取り消し（Undo）が可能であること
- 取り消した操作のやり直し（Redo）が可能であること
- Undo/Redo の履歴数は設定可能とする（デフォルト値は基本設計で決定）

### 3.7 プロジェクト管理

ワイヤーフレームプロジェクト全体の管理機能。

#### 3.7.1 複数ページ管理

- 1つのプロジェクト内で複数のページ（画面）を管理できること
- ページの追加・削除・並び替えが可能であること
- ページ間の移動がスムーズに行えること

#### 3.7.2 ページ間リンク（画面遷移の定義）

- ワイヤーフレーム上の要素にリンク先のページを設定できること
- 画面遷移の流れを確認できるプレビュー機能として、以下の2種類を提供する
  - **クリッカブルプロトタイプ**: リンクが設定された要素をクリックすると、実際にリンク先ページに遷移するインタラクティブなプレビュー
  - **遷移図ダイアグラム**: ページ間の遷移関係を俯瞰的に確認できるフロー図形式のビュー

#### 3.7.3 テンプレート機能

- **自作テンプレート**: 作成したワイヤーフレーム（ページ単位またはコンポーネント単位）をテンプレートとして保存し、再利用できること
- テンプレートの保存・読み込み・削除が可能であること

### 3.8 エクスポート

ワイヤーフレームを各種形式で出力する機能。

#### 3.8.1 画像エクスポート

| 形式 | 説明 |
|------|------|
| PNG | ラスタ画像として出力 |
| SVG | ベクタ画像として出力 |

#### 3.8.2 コードエクスポート

| 形式 | 説明 |
|------|------|
| HTML/CSS | Webブラウザで表示可能なHTML/CSSコードとして出力 |
| WPF XAML | WPF（.NET Framework / .NET）向けXAMLコードとして出力 |
| WinUI 3 XAML | WinUI 3（Windows App SDK）向けXAMLコードとして出力 |
| .NET MAUI XAML | .NET MAUI 向けXAMLコードとして出力 |
| Avalonia UI XAML | Avalonia UI 向けXAMLコードとして出力 |

**品質期待レベル: レイアウトの忠実な再現**

コードエクスポートは、ワイヤーフレーム上のレイアウト（要素の配置、サイズ、親子構造、スペーシング等）を可能な限り忠実に再現したコードを出力することを目標とする。各ターゲットのレイアウトモデルの差異を考慮し、最も適切なレイアウト手法を選択して出力すること。

※ 各ターゲット間のレイアウトモデルの差異（CSS Flexbox/Grid vs WPF StackPanel/Grid vs 等）への具体的な変換ルールは基本設計で策定する。

#### 3.8.3 エクスポート共通要件

- ページ単位でのエクスポートが可能であること
- プロジェクト全体の一括エクスポートが可能であること
- エクスポート時の品質・解像度の設定が可能であること（画像形式の場合）

## 4. 非機能要件

### 4.1 対応環境

- 基本設計フェーズで確定する（Webアプリの場合、対応ブラウザ等）

### 4.2 パフォーマンス

- 要素数が多い場合でもスムーズな操作が可能であること
- 具体的な数値目標は基本設計フェーズで設定する

### 4.3 データ保存

#### 4.3.1 保存形式

- **独自形式**: プロジェクトデータは独自のファイル形式で保存する
- 独自形式のフォーマット仕様は基本設計で策定する

#### 4.3.2 保存要件

以下の保存に関する要件を満たすこと。具体的な保存方法の実現手段はアプリケーション形態の決定後に基本設計で策定する。

- プロジェクトデータを永続的に保存できること
- 保存したプロジェクトデータを読み込んで編集を再開できること
- 独自形式ファイルとしてのエクスポート（書き出し）・インポート（読み込み）が可能であること
- **オフライン環境でも保存・読み込みが可能であること**（必須）

#### 4.3.3 自動保存

- 自動保存機能の要否・仕様は基本設計で検討する

### 4.4 操作性

- キーボードショートカットによる主要操作が可能であること
- ショートカットキーの一覧は基本設計で定義する

### 4.5 UI表示言語

- **日本語のみ**対応とする

### 4.6 技術スタック

- 使用する技術スタック（プログラミング言語、フレームワーク、キャンバス描画技術等）は基本設計フェーズで選定する

## 5. 段階的拡張方針

本ツールは段階的に拡張する方針とする。フェーズの区切りと各フェーズの対象機能は基本設計フェーズで策定する。

### 5.1 フェーズ分けの方針（案）

| フェーズ | 対象機能（案） |
|----------|----------------|
| フェーズ1 | キャンバス操作、基本図形要素、要素構造・コンテナ管理の基本機能 |
| フェーズ2 | UIコンポーネント、レイアウト・編集の全機能 |
| フェーズ3 | プロジェクト管理、テンプレート機能 |
| フェーズ4 | エクスポート機能（画像＋HTML/CSS） |
| フェーズ5 | エクスポート機能（XAML系） |

※ 上記は方針案であり、具体的なフェーズ分けは基本設計で確定する。

### 5.2 プロジェクト進行上の留意事項

- **コードエクスポートの実現性検証（PoC）**: コードエクスポート機能（3.8.2）は、5つのターゲット（HTML/CSS、WPF、WinUI 3、.NET MAUI、Avalonia UI）すべてで「レイアウトの忠実な再現」を目標としており、本ツール開発全体の中で最もリスクの高い部分である。各ターゲットのレイアウトモデルは互いにかなり異なるため、基本設計の段階で「内部レイアウトモデル → 各ターゲットへの変換」の実現性を早期にプロトタイプ検証（PoC）することを推奨する。
