## 1. 概要

### 1.1 本設計書の範囲

実装プロジェクトにおけるドキュメント類（ユーザーマニュアル、開発者ガイド等）の体系を定義する。

本設計書が定義する範囲:

- ドキュメントの分類と一覧
- VitePress を用いたドキュメントサイトの構成
- アプリ内ヘルプと外部ドキュメントの使い分け方針
- 開発フェーズに合わせた段階的整備計画
- 将来の公開対応に向けた準備事項

本設計書の範囲外:

- 各ドキュメントの具体的な記述内容 — 実装プロジェクトで作成
- アプリ内ヘルプの実装詳細 — 横断07（画面レイアウト設計）を参照

### 1.2 関連する設計書からの入力

| 入力元 | 主な入力内容 |
|--------|------------|
| 横断07（画面レイアウト設計） | アプリ内ヘルプの設計済み要素（ヘルプメニュー、ツールチップ、ショートカット一覧ダイアログ） |
| テーマ4（操作・インタラクション設計） | キーボードショートカット一覧（60+、6カテゴリ）、カスタマイズ方針 |
| テーマ2（データ設計・永続化） | .wfp ファイル形式仕様（ドキュメント化対象） |
| テーマ5（エクスポート・変換設計） | エクスポート変換ルール（ドキュメント化対象） |
| テーマ6（開発ロードマップ） | 5フェーズ + PoC の定義（段階的整備計画の基準） |
| 横断08（モジュール構成） | プロジェクトルートのディレクトリ構成（docs/ の配置先） |
| テーマ1（アプリケーション基盤） | 技術スタック（Vite ベース → VitePress との親和性） |

## 2. ドキュメント分類と一覧

### 2.1 ユーザー向けドキュメント

ツールの使い方を説明するドキュメント。VitePress サイトの `guide/` セクションに配置する。

| ドキュメント名 | 内容 | 優先度 | 作成タイミング |
|--------------|------|--------|-------------|
| クイックスタートガイド | 初回起動から基本的なワイヤーフレーム作成・保存までの手順 | 高 | フェーズ1完了時 |
| 機能リファレンス（複数ページ） | 各機能の詳細な使い方。基本図形、コンテナ・レイアウト、UIコンポーネント、プロジェクト管理の4分野 | 中 | 各フェーズ完了時に対応分野を追記 |
| エクスポートガイド（複数ページ） | 各エクスポートターゲット（画像、HTML/CSS、XAML）の使い方・制約事項 | 中 | フェーズ4-5完了時 |
| FAQ / トラブルシューティング | よくある問題と解決方法 | 低 | 公開準備時 |

### 2.2 アプリ内ヘルプ（設計済み要素の確認）

横断07（画面レイアウト設計）およびテーマ4（操作・インタラクション設計）で既に設計済みのアプリ内ヘルプ要素を確認する。

| ヘルプ要素 | 内容 | 設計書の参照先 |
|-----------|------|-------------|
| ツールチップ（`v-tooltip`） | ツールバーボタンの機能名とショートカットキーを表示 | 横断07 セクション4.2 |
| キーボードショートカット一覧ダイアログ | 60+ のショートカットをカテゴリ別に表示 | 横断07 セクション3.1（ヘルプメニュー）、テーマ4 セクション8 |
| バージョン情報ダイアログ | アプリケーションのバージョン情報を表示 | 横断07 セクション3.1（ヘルプメニュー） |
| ステータスバー情報 | ズーム率、選択要素情報、キャンバスサイズを常時表示 | 横断07 セクション2 |

## 3. ドキュメントサイト構成（VitePress）

### 3.1 VitePress 選定理由

| 観点 | 説明 |
|------|------|
| 技術スタックとの親和性 | 本ツールは Vite ベースの SPA+PWA であり、VitePress も Vite ベース。ビルドツールチェーンの知識を共有できる |
| コンテンツ形式 | Markdown ファイルをそのままページとして利用。特殊な記法を学習する必要がない |
| 標準機能 | 全文検索、サイドバーナビゲーション、レスポンシブデザイン、ダークモードが標準装備 |
| アプリとの連携 | PWA アプリのヘルプメニューから VitePress サイトへリンクで遷移可能 |
| 保守コスト | Vite エコシステムの一部として継続的にメンテナンスされている。Vue 公式ドキュメントでも採用 |

### 3.2 実装プロジェクト内の配置

```
IntuitiveD4UI/                         # 実装プロジェクトルート
├── README.md                          # プロジェクト概要・セットアップ手順
├── CHANGELOG.md                       # 変更履歴
├── LICENSE                            # ライセンス
├── docs/                              # VitePress ドキュメントサイト
│   ├── .vitepress/
│   │   └── config.ts                  # VitePress 設定（サイトタイトル、ナビ、サイドバー等）
│   ├── index.md                       # トップページ
│   ├── guide/                         # ユーザー向けガイド
│   │   ├── quick-start.md             # クイックスタートガイド
│   │   ├── basic-shapes.md            # 基本図形の使い方
│   │   ├── containers-layout.md       # コンテナとレイアウト
│   │   ├── ui-components.md           # UIコンポーネント
│   │   ├── project-management.md      # プロジェクト管理
│   │   └── export/                    # エクスポート機能
│   │       ├── image-export.md        # PNG/SVG エクスポート
│   │       ├── html-css-export.md     # HTML/CSS エクスポート
│   │       └── xaml-export.md         # XAML エクスポート（4ターゲット統合）
│   ├── reference/                     # 開発者向けリファレンス
│   │   ├── architecture.md            # アーキテクチャ概要
│   │   ├── file-format-spec.md        # .wfp ファイル形式仕様
│   │   └── export-rules.md            # エクスポート変換ルール
│   ├── faq.md                         # FAQ / トラブルシューティング（公開準備時）
│   └── public/                        # ドキュメント用静的ファイル
│       └── images/                    # スクリーンショット・図表
├── src/                               # アプリケーションソースコード
│   └── ...
├── package.json                       # VitePress の依存を devDependencies に含める
└── ...
```

**配置方針:**

- `docs/` はプロジェクトルート直下に配置し、VitePress の標準構成に従う
- アプリケーションの `src/` とは完全に独立したディレクトリとして管理する
- `guide/` にユーザー向けドキュメント、`reference/` に開発者向けドキュメントを配置し、対象読者ごとに分離する
- `guide/export/` はエクスポート関連のドキュメントが複数あるためサブディレクトリ化する
- `index.md` はサイトのトップページとして、ツール概要と各セクション（ガイド、リファレンス）へのナビゲーションを提供する
- README.md、CHANGELOG.md、LICENSE はプロジェクトルート直下に配置（OSS/公開プロジェクトの慣習）

### 3.3 サイトナビゲーション構成

VitePress の `.vitepress/config.ts` で定義するナビゲーション構成。

**トップナビゲーション:**

| ナビ項目 | リンク先 |
|---------|---------|
| ガイド | `/guide/quick-start` |
| リファレンス | `/reference/architecture` |
| FAQ | `/faq`（公開準備時に追加） |

**サイドバー構成:**

```
ガイド
├── クイックスタート          → /guide/quick-start
├── 基本図形                 → /guide/basic-shapes
├── コンテナとレイアウト      → /guide/containers-layout
├── UIコンポーネント          → /guide/ui-components
├── プロジェクト管理          → /guide/project-management
└── エクスポート
    ├── 画像エクスポート      → /guide/export/image-export
    ├── HTML/CSS エクスポート → /guide/export/html-css-export
    └── XAML エクスポート     → /guide/export/xaml-export

リファレンス
├── アーキテクチャ概要        → /reference/architecture
├── .wfp ファイル形式仕様     → /reference/file-format-spec
└── エクスポート変換ルール    → /reference/export-rules
```

サイドバーの項目はフェーズの進行に合わせて段階的に追加する。未作成のページへのリンクは含めない。

## 4. アプリ内ヘルプと外部ドキュメントの使い分け方針

### 4.1 使い分けの原則

| 情報の性格 | 配置先 | 理由 |
|-----------|--------|------|
| 操作中に即座に必要な情報 | アプリ内ヘルプ | ツール切替や操作の流れを中断させない |
| 操作名・概要程度の情報（1-2行） | アプリ内（ツールチップ） | インラインで十分伝わる |
| 学習・理解のための情報 | VitePress ドキュメントサイト | じっくり読む情報はアプリ外で提供 |
| リファレンス的に検索したい情報 | VitePress ドキュメントサイト | VitePress の全文検索機能を活用 |

### 4.2 具体的な振り分け表

| 情報 | 配置先 | 形式 |
|------|--------|------|
| 「矩形ツールは何をするか」 | アプリ内: ツールチップ | `v-tooltip`: "矩形を配置 (R)" |
| 「Ctrl+G でグループ化」 | アプリ内: ショートカット一覧ダイアログ + メニュー横の表記 | ダイアログ内テーブル |
| 「コンテナとグループの違い」 | 外部: `guide/containers-layout.md` | 概念説明 + 図表 |
| 「Flex レイアウトの各プロパティの効果」 | 外部: `guide/containers-layout.md` | スクリーンショット付き解説 |
| 「HTML/CSS エクスポートで再現されないケース」 | 外部: `guide/export/html-css-export.md` | 制約一覧 + 回避策 |
| 「XAML ターゲット間の差異」 | 外部: `guide/export/xaml-export.md` | 比較表 |
| 「.wfp ファイルの内部構造」 | 外部: `reference/file-format-spec.md` | JSON スキーマ解説 |
| 「アーキテクチャの全体像」 | 外部: `reference/architecture.md` | 図 + 依存関係説明 |

**ショートカット一覧の二重管理防止:**

アプリ内のショートカットマッピングテーブル（テーマ4 セクション8.3 で定義されたマッピングテーブル管理方式）を唯一の信頼元（Single Source of Truth）とする。VitePress ドキュメントサイト側には「キーボードショートカットの一覧はアプリ内のヘルプメニュー → キーボードショートカット一覧で確認できます」と記載し、一覧表の重複管理を避ける。

## 6. 将来の公開対応

### 6.1 今から準備しておく事項

初期段階から以下を意識することで、将来の公開時の追加作業を最小化する。

| 準備事項 | 具体的な対応 |
|---------|------------|
| ディレクトリ構成を最初から確定する | セクション3.2 の構成を初期から配置。後からの再構成はリンク切れのリスクがある |
| README.md をきちんと書く | 開発中から整備しておくと、公開時にゼロから書く負担がない |
| CHANGELOG.md をフェーズごとに記録する | 後からまとめて書くのは変更内容の追跡が困難。[Keep a Changelog](https://keepachangelog.com/) 形式に従う |
| 用語を統一する | 基本設計書 00-overview.md セクション5（用語集）の定義を基準とする |
| 記述言語は日本語で統一する | UI 表示言語が日本語のみ（要件定義書で決定済み）であり、ドキュメントも日本語が自然。将来の国際化時には翻訳で対応 |

### 6.2 公開時に追加で必要になる事項

現時点では不要だが、公開・他者利用時に必要になるもの。

| 必要なもの | 内容 |
|-----------|------|
| CONTRIBUTING.md | コーディング規約、PR方法、Issue報告方法 |
| FAQ / トラブルシューティング | ユーザーからの問い合わせ削減のための事前情報 |
| GitHub Pages デプロイ設定 | VitePress サイトを Web 上に公開するための CI/CD パイプライン |
| ヘルプメニューへのリンク追加 | アプリ内ヘルプメニューに「ユーザーガイド」項目を追加し、VitePress サイトへ遷移する機能 |
| ドキュメントのスクリーンショット充実 | 操作手順の図解を追加。テキストだけでは伝わりにくい操作の説明に必要 |
| ライセンス選定の確定 | 公開形態（OSS / フリーウェア等）に応じたライセンスの選定 |
