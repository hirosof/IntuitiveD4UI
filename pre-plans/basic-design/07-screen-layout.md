## 1. 概要

### 1.1 本設計書の範囲

本設計書は、Intuitive D4UI の画面全体のレイアウト構成を定義する横断的設計書である。テーマ1〜5の設計書で分散して決定・言及された画面UI要素を統合し、以下を規定する。

- 画面全体のエリア分割とパネル構成
- メニューバー・ツールバーの構成
- 各サイドバーの内容と操作
- キャンバスエリアのUI要素配置
- コンテキストメニューの構成
- z-index管理方針
- CSS方針とUIコンポーネントライブラリの技術選定（テーマ1から委任された未決定事項の確定）

### 1.2 前提とする設計書からの入力

| 設計書 | 入力事項 |
|--------|---------|
| テーマ1（01-application-architecture.md） | Vue.js + Vite 技術スタック、CSS方針/UIライブラリの委任 |
| テーマ2（02-data-design.md） | ElementNode.name（ツリービュー表示名）、データモデル構造 |
| テーマ3（03-canvas-rendering.md） | 3層構成（gridLayer, elementLayer, overlayLayer）、DOMオーバーレイ、キャンバスサイズプリセット |
| テーマ4（04-interaction-design.md） | プロパティパネルの表示仕様、パンくずリスト、整列・均等配置UI（委譲）、キーボードショートカット一覧、コンテキストメニュー項目 |
| テーマ5（05-export-design.md） | エクスポートダイアログ構成 |

## 2. 全体画面レイアウト

### 2.1 レイアウト構成図

```
┌──────────────────────────────────────────────────────────────┐
│ ファイル | 編集 | 表示 | 要素 | ヘルプ                        │ ← メニューバー (32px)
├──────────────────────────────────────────────────────────────┤
│ [↖選択] │ [Undo][Redo] │ [ズーム −/数値/+][フィット] │ [整列]│ ← ツールバー (40px)
├────────┬──────────────────────────────────┬─────────────────┤
│[ページ] │  パンくずリスト（条件付き表示）    │ レイヤーツリー   │
│[ツール] ├──────────────────────────────────┤ （上部）         │
│ ────── │                                  ├─────────────────┤
│        │                                  │ プロパティパネル  │
│ タブの  │        キャンバスエリア              │ （下部）         │
│ 内容   │                                  │                 │
│        │                                  │ 5セクション      │
│        │                                  │ アコーディオン    │
│ 240px  │         （残余スペース）            │ 280px           │
├────────┼──────────────────────────────────┼─────────────────┤
│                 ステータスバー (24px)                         │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 各エリアの定義と Vuetify コンポーネント対応

| エリア | Vuetify コンポーネント | 説明 |
|--------|---------------------|------|
| アプリケーションルート | `v-app` | Vuetify アプリケーションのルート要素 |
| メニューバー | `v-app-bar` + `v-menu` | アプリケーション上端に固定。メニュー項目はドロップダウン |
| ツールバー | `v-toolbar` + `v-btn` + `v-tooltip` | メニューバー直下に固定。頻用操作のアイコンボタン群 |
| 左サイドバー | `v-navigation-drawer`（location="left"） | タブ切り替え式。ページパネルとツールボックスパネル |
| キャンバスエリア | `v-main` 内の Canvas コンテナ | Konva.Stage を配置。DOMオーバーレイを含む |
| 右サイドバー | `v-navigation-drawer`（location="right"） | 上下分割。レイヤーツリーとプロパティパネル |
| ステータスバー | `v-footer`（app） | ズーム倍率表示、キャンバスサイズ表示等 |

### 2.3 パネル幅とリサイズ仕様

| パネル | デフォルト幅/高さ | 最小 | 最大 | リサイズ |
|--------|----------------|------|------|---------|
| 左サイドバー | 240px | 180px | 400px | ドラッグリサイズ可 |
| 右サイドバー | 280px | 220px | 440px | ドラッグリサイズ可 |
| メニューバー | 高さ 32px | — | — | 固定 |
| ツールバー | 高さ 40px | — | — | 固定 |
| ステータスバー | 高さ 24px | — | — | 固定 |
| キャンバスエリア | 残余スペース | — | — | 自動（CSS flex: 1） |

**リサイズ操作**:
- ドラッグハンドル（4px 幅）を左右サイドバーの内側境界に配置
- ホバー時にカーソルが `col-resize` に変化し、ハンドルの色が変化
- ドラッグ中はリアルタイムでパネル幅が変化
- リサイズ操作は自前実装（Vuetify 標準にはパネル分割リサイズ機能がないため）

### 2.4 パネル折りたたみ仕様

| パネル | 折りたたみ | 方式 |
|--------|-----------|------|
| 左サイドバー | 可 | 幅 0px + 展開ボタン表示 |
| 右サイドバー | 可 | 幅 0px + 展開ボタン表示 |
| メニューバー | 不可 | 常時表示 |
| ツールバー | 不可 | 常時表示 |
| ステータスバー | 不可 | 常時表示 |

- **折りたたみ時**: サイドバーの代わりに 16px 幅の展開ボタン（矢印アイコン）を表示
- **キャンバスエリア**: 折りたたまれたサイドバーの分だけ自動拡張
- **アニメーション**: 折りたたみ/展開時に 200ms ease-out のトランジション
- **状態の保存**: 折りたたみ状態は `localStorage` に保存し、次回起動時に復元
- **メニューからのアクセス**: 「表示」メニューから左サイドバー/右サイドバーの表示/非表示をトグル可能

## 4. ツールバー

### 4.1 グループ構成

ツールバーは `v-toolbar`（density="compact"）で構成し、左から右に以下の4グループを配置する。各グループはセパレータ（`v-divider`、vertical）で区切る。

```
[↖ 選択] │ [↶ Undo][↷ Redo] │ [− ズームアウト][100%][+ ズームイン][⊞ フィット] │ [整列ボタン群]
```

| グループ | 含まれるボタン | 備考 |
|---------|-------------|------|
| **選択ツール** | 選択ツール（↖） | 現在アクティブなツールモードを反映（セクション5.4参照） |
| **Undo/Redo** | Undo, Redo | スタック空時はグレーアウト |
| **ズーム制御** | ズームアウト, ズーム倍率表示, ズームイン, フィット表示 | ズーム倍率は数値表示（例: 100%） |
| **整列・配置** | 左揃え, 右揃え, 上揃え, 下揃え, 水平中央, 垂直中央, 水平均等, 垂直均等 | 右寄せ配置 |

### 4.2 整列・均等配置ボタン

テーマ4 セクション7.3から委譲された仕様に基づく。

| ボタン | アイコン | 有効条件 | 動作 |
|--------|---------|---------|------|
| 左揃え | align-left | 2要素以上選択 + Free レイアウト | 全選択要素の左端を最左に揃える |
| 右揃え | align-right | 同上 | 全選択要素の右端を最右に揃える |
| 上揃え | align-top | 同上 | 全選択要素の上端を最上に揃える |
| 下揃え | align-bottom | 同上 | 全選択要素の下端を最下に揃える |
| 水平中央揃え | align-center-h | 同上 | 全選択要素の水平中心を中央に揃える |
| 垂直中央揃え | align-center-v | 同上 | 全選択要素の垂直中心を中央に揃える |
| 水平均等配置 | distribute-h | 3要素以上選択 + Free レイアウト | 選択要素間の水平間隔を均等化 |
| 垂直均等配置 | distribute-v | 同上 | 選択要素間の垂直間隔を均等化 |

- 有効条件を満たさない場合はグレーアウト
- 各ボタンにはツールチップ（`v-tooltip`）で操作名を表示

## 5. 左サイドバー

左サイドバーは `v-navigation-drawer`（permanent, location="left"）で構成し、タブ切り替え（`v-tabs` + `v-window`）で「ページ」と「ツールボックス」を切り替える。

### 5.1 タブ構成

```
┌──────────────────┐
│ [ページ] [ツール]  │ ← v-tabs
├──────────────────┤
│                  │
│  タブに応じた     │ ← v-window / v-window-item
│  コンテンツ       │
│                  │
└──────────────────┘
```

### 5.2 ページタブ

アクティブなページの管理と切り替えを行う。

| 要素 | 説明 |
|------|------|
| ページリスト | 各ページ名をリスト表示。選択中ページはハイライト |
| ページ追加ボタン | リスト下部に「+」ボタン。クリックで新規ページを追加 |
| ページ操作 | 右クリックコンテキストメニューで削除・複製・名前変更 |

- ページ名はクリックで切り替え、ダブルクリックで名前変更
- ページ数が多い場合は縦スクロール

### 5.3 ツールボックスタブ

要素の配置ツールを提供する。「基本図形」と「UIコンポーネント」をアコーディオン（`v-expansion-panels`）で切り替える。

```
┌──────────────────┐
│ ▼ 基本図形        │ ← v-expansion-panel
│   □ 矩形         │
│   ○ 楕円         │
│   ─ 線           │
│   → 矢印         │
│   T テキスト      │
│   🖼 画像         │
│   ▢ コンテナ      │
│ ▶ UIコンポーネント │ ← v-expansion-panel（折りたたみ時）
└──────────────────┘
```

#### 基本図形

| 項目 | ElementType | 説明 |
|------|------------|------|
| 矩形 | `rectangle` | 角丸対応の矩形 |
| 楕円 | `ellipse` | 楕円 |
| 線 | `line` | 直線 |
| 矢印 | `arrow` | 矢印付き線 |
| テキスト | `text` | テキスト要素 |
| 画像 | `image-placeholder` | 画像プレースホルダー |
| コンテナ | `container` | レイアウト機能付きコンテナ |

#### UIコンポーネント

| 項目 | ElementType | 説明 |
|------|------------|------|
| ボタン | `ui-button` | ボタン |
| 入力欄 | `ui-input` | テキスト入力フィールド |
| ドロップダウン | `ui-dropdown` | ドロップダウンセレクト |
| チェックボックス | `ui-checkbox` | チェックボックス |
| ラジオボタン | `ui-radio` | ラジオボタン |

- 各項目にはアイコン + 要素名を表示
- **group はパレットに含めない**: グループ化は複数要素選択 + Ctrl+G で作成（テーマ4 セクション5.1.1）

### 5.4 ツールモード方式

要素パレットから図形/UIコンポーネントを選択すると「配置ツールモード」に入り、キャンバス上で連続的に要素を配置できる。

| 状態 | 説明 |
|------|------|
| **選択ツールモード**（デフォルト） | 通常の選択・移動・リサイズ操作 |
| **配置ツールモード** | キャンバス上でクリック/ドラッグすると要素を配置。配置後もモードを維持 |

**モード切り替え**:

| 操作 | 結果 |
|------|------|
| ツールボックスで図形をクリック | 配置ツールモードに入る。カーソルがクロスヘア（+）に変化 |
| キャンバス上でクリック/ドラッグ | 要素を配置。配置ツールモードは維持（連続配置可能） |
| ESC キー | 選択ツールモードに戻る |
| 右クリック | 選択ツールモードに戻る |
| ツールバーの選択ツールボタン（↖）をクリック | 選択ツールモードに戻る |

- ツールバーの選択ツールボタン（↖）のアイコンは、現在のモード状態を反映する
- 配置ツールモード中は、選択ツールボタンが非アクティブ状態（押されていない見た目）になる

## 6. キャンバスエリア

### 6.1 3層構成との関係

キャンバスエリアは `v-main` 内に配置され、Konva.Stage がマウントされる。テーマ3で定義された3層構成（gridLayer, elementLayer, overlayLayer）はキャンバス内部の描画レイヤーであり、画面レイアウトのz-index管理とは独立している。

```
v-main
└── canvas-container (div)
    ├── Konva.Stage (Canvas 要素)
    │   ├── gridLayer
    │   ├── elementLayer
    │   └── overlayLayer
    └── DOM オーバーレイ (div, position: absolute)
        ├── テキスト編集用 <textarea>
        └── その他のDOM要素
```

### 6.2 DOMオーバーレイの配置方式

テーマ3 セクション2.5で定義された DOMオーバーレイ（テキスト編集用 `<textarea>` 等）は、Canvas 要素と同じ親コンテナ内に `position: absolute` で配置する。

- Canvas のズーム・パン変換と同期して位置・スケールを調整
- z-index は Canvas 要素より上に配置（セクション9.1参照）

### 6.3 パンくずリスト

ドリルダウン状態（`selectionContext !== null`）の場合にのみ表示するパンくずリスト。`v-breadcrumbs` で実装する。

| 項目 | 仕様 |
|------|------|
| 配置 | キャンバスエリア上端、ツールバー直下 |
| 表示条件 | `selectionContext !== null`（ドリルダウン中）の場合のみ表示 |
| 非表示時 | 高さ 0px。キャンバスがその分拡張 |
| 表示時 | 高さ 28px。キャンバスの上にスペースを確保して表示 |
| 形式 | `Root > Container A > Container B`（各階層はクリック可能） |
| 操作 | 各階層をクリックで任意の階層にジャンプ（テーマ4 セクション2.6.3） |

### 6.4 ズーム/パンコントロール

ステータスバーにズーム倍率の数値表示を配置する（ツールバーのズーム制御と連動）。

## 7. 右サイドバー

右サイドバーは `v-navigation-drawer`（permanent, location="right"）で構成し、上下分割でレイヤーツリー（上部）とプロパティパネル（下部）を配置する。

### 7.1 上下分割

```
┌──────────────────┐
│ レイヤーツリー     │ ← 上部（デフォルト高さ: パネル高さの40%、最小 80px）
│                  │
├──────────────────┤ ← ドラッグリサイズ可能な境界
│ プロパティパネル   │ ← 下部（残余高さ）
│                  │
│                  │
└──────────────────┘
```

- ドラッグハンドル（4px 高さ）で上下パネルの高さ比率を調整可能
- リサイズ操作は自前実装

### 7.2 レイヤーツリー

`v-treeview` を使用して、現在のページの要素をツリー構造で表示する。

#### ツリーノードの表示

各ノードには以下の情報を表示する:

| 要素 | 説明 |
|------|------|
| アイコン | ElementType に応じたアイコン |
| 要素名 | `ElementNode.name`（ダブルクリックで編集可能） |
| 可視性トグル | 目アイコン（`visible` のトグル） |
| ロックトグル | 鍵アイコン（`locked` のトグル） |

#### 要素の移動操作

ツリービュー上での要素の移動は、D&D ではなくコンテキストメニューおよびボタン操作で行う。

| 操作 | 説明 | データ操作 |
|------|------|-----------|
| ↑ 上に移動 | 同一親内で1つ上（`childrenIds` 内で1つ前）に移動 | `childrenIds` の順序変更 |
| ↓ 下に移動 | 同一親内で1つ下（`childrenIds` 内で1つ後）に移動 | `childrenIds` の順序変更 |
| → 中に入れる | 直下の兄弟コンテナ/グループの子要素として移動 | `parentId` と `childrenIds` の更新 |
| ← 外に出す | 親のコンテナ/グループから1つ上の階層に移動 | `parentId` と `childrenIds` の更新 |
| ⇈ 先頭に移動 | 同一親内で先頭に移動 | `childrenIds` の順序変更 |
| ⇊ 末尾に移動 | 同一親内で末尾に移動 | `childrenIds` の順序変更 |

- データ操作は、キャンバス上のD&D（テーマ4 セクション3.1〜3.3）と同じロジック（`parentId` / `childrenIds` の更新）を使用する
- 「中に入れる」操作時はネスト深さチェック（推奨8階層/上限16階層）を適用
- 要素を複数選択している場合は、選択中の全要素に対してまとめて操作を適用
- キャンバス上のD&D機能は引き続き利用可能であり、直感的な要素移動はキャンバス上で行う設計

#### レイヤーツリーの選択連動

- ツリーのノードをクリック → キャンバス上の対応する要素が選択状態になる
- キャンバス上で要素を選択 → ツリーの対応ノードがハイライトされる
- 複数選択（Shift+クリック）にも対応

#### パフォーマンス考慮

- 要素数が多い場合に備え、仮想スクロール（virtual scroll）を導入し、表示領域内のノードのみをDOMに描画する

### 7.3 プロパティパネル

5セクションのアコーディオン（`v-expansion-panels`、variant="accordion"）で構成する。

#### セクション構成

| セクション | 含まれるプロパティ | 表示条件 |
|-----------|------------------|---------|
| **1. 要素情報** | 要素名（編集可能）、type 表示、ID 表示 | 常に表示 |
| **2. 位置・サイズ** | x, y, width, height, rotation | 常に表示 |
| **3. 外観** | opacity, fill（色）, stroke（色/幅）, cornerRadius, visible, locked | 常に表示（type 依存で項目を出し分け） |
| **4. レイアウト** | layout 種別, padding（4方向）, overflow, margin（4方向） | container 選択時のみ表示 |
| **5. リンク** | リンク先ページ選択 | 常に表示 |

#### 各セクションの入力UI

| プロパティ | Vuetify コンポーネント | 備考 |
|-----------|---------------------|------|
| 要素名 | `v-text-field`（variant="outlined", density="compact"） | |
| x, y, width, height, rotation | `v-number-input` | |
| opacity | `v-slider` + `v-number-input` | 0〜1 の範囲 |
| fill, stroke 色 | `v-color-picker` | スウォッチ付き（`show-swatches`） |
| stroke 幅 | `v-number-input` | |
| cornerRadius | `v-number-input` | rectangle 選択時のみ |
| visible, locked | `v-switch`（density="compact"） | |
| layout.type | `v-select` | Free / Flex / Grid |
| layout.direction | `v-select` | row / column（Flex 時のみ） |
| layout.wrap | `v-checkbox` | Flex 時のみ |
| layout.gap | `v-number-input` | Flex/Grid 時のみ |
| layout.columns, rows | `v-number-input` | Grid 時のみ |
| padding（4方向） | `v-number-input` × 4 | リンクボタンで一括/個別切り替え |
| overflow | `v-select` | visible / clip |
| margin（4方向） | `v-number-input` × 4 | |
| リンク先ページ | `v-select` | 同一プロジェクト内のページ一覧 |

### 7.4 選択状態別の表示パターン

テーマ4 セクション2.1の選択モデルに基づく表示パターン。

| 状態 | プロパティパネルの表示 |
|------|---------------------|
| `selectedIds` が空（未選択） | プレースホルダー表示（「要素を選択してください」）またはページ情報（ページ名、キャンバスサイズ） |
| `selectedIds` が1件（単一選択） | 選択要素の全プロパティを表示 |
| `selectedIds` が複数（複数選択） | 共通プロパティのみ表示。値が異なるフィールドは「—」（混在表示） |
| ロック中の要素 | 全フィールドを読み取り専用（グレーアウト）。セクションは表示するが編集不可 |

### 7.5 レイアウトセクションの動的表示

レイアウトセクション（セクション4）は container 選択時のみ表示され、選択中の `layout.type` に応じて表示するフィールドが変化する。

| layout.type | 表示するフィールド |
|------------|-----------------|
| Free | （追加フィールドなし） |
| Flex | direction, wrap, gap |
| Grid | columns, rows, columnSizes, rowSizes, gap |

## 9. z-index 管理

### 9.1 キャンバス関連の自前 z-index 定義

キャンバスエリア内のDOM要素の重なり順序を以下のように定義する。

| z-index | レイヤー名 | 対象要素 |
|---------|----------|---------|
| 0 | base | Konva.Stage（Canvas 要素） |
| 10 | canvas-overlay | DOMオーバーレイ（テキスト編集 textarea） |
| 20 | breadcrumb | パンくずリスト |

- 10刻みにすることで、将来の挿入余地を確保

### 9.2 Vuetify デフォルト体系との共存方針

UIパネル（サイドバー、ツールバー、メニュー、ダイアログ等）の z-index は Vuetify のデフォルト体系に委任する。

| 要素 | Vuetify のデフォルト z-index |
|------|---------------------------|
| `v-navigation-drawer` | 1006 |
| `v-app-bar` | 1006 |
| `v-toolbar` | 自動 |
| `v-menu` | 2000 台 |
| `v-dialog` | 2400 台 |
| `v-snackbar` | 2500 台 |

- キャンバス関連の自前 z-index（0〜20）と Vuetify の z-index（1000 台以上）は値域が離れているため、競合しない
- カスタム z-index が必要な場合のみ、Vuetify のテーマ設定（`$navigation-drawer-z` 等の SASS 変数）でオーバーライドする
- モーダルダイアログ（`v-dialog`）表示時は、Vuetify の標準 overlay 機能により背景要素へのインタラクションが自動でブロックされる

## 10. 技術選定

### 10.1 CSS方針

| 項目 | 決定内容 |
|------|---------|
| ユーティリティCSS | **不使用**（UnoCSS / Tailwind CSS 等は採用しない） |
| メインのスタイリング | Vuetify のスタイルシステム（Material Design テーマ） |
| コンポーネント固有スタイル | Vue SFC `<style scoped>` |
| テーマカスタマイズ | Vuetify の SASS 変数によるテーマ設定 |

**使い分け方針**:
- **レイアウト・スペーシング**: Vuetify のクラス（`d-flex`, `pa-2`, `ma-1` 等）および CSS Grid/Flexbox
- **コンポーネント固有の見た目**: Vue SFC `<style scoped>` で記述
- **カラー・フォント等のテーマ**: Vuetify のテーマ設定で一元管理

### 10.2 UIコンポーネントライブラリ: Vuetify

| 項目 | 決定内容 |
|------|---------|
| ライブラリ | **Vuetify**（Vue 3 対応、Material Design ベース） |
| 選定理由 | 豊富なコンポーネント群、充実したカラーピッカー、大規模コミュニティ、TypeScript 対応 |

| コンポーネント | 理由 | 実装方針 |
