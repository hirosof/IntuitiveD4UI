## 1. 基本設計の目的

要件定義書 v0.3.0 で定義された「Intuitive D4UI」の要件を具体的な設計に落とし込み、開発に必要な技術的決定と詳細仕様をまとめる。

```

  ├─→ 03-canvas-rendering（キャンバス）
```

## 4. 技術スタック概要

テーマ1（`01-application-architecture.md`）で決定済み。

| カテゴリ | 技術 |
|----------|------|
| アプリ形態 | SPA + PWA |
| 対応ブラウザ | Chromium系（Chrome, Edge） |
| 描画技術 | Canvas 2D（Konva.js + vue-konva）+ DOMオーバーレイ |
| 言語 | TypeScript |
| フレームワーク | Vue.js（Vue 3, Composition API） |
| 状態管理 | Pinia |
| ビルドツール | Vite |
| テスト | Vitest + Playwright |
| UIコンポーネント | Vuetify |
| CSS方針 | Vuetifyスタイルシステム + Vue SFC scoped CSS |

## 5. 用語集

| 用語 | 定義 |
|------|------|
| コンテナ | 子要素を内包できる要素。HTML の `<div>` に相当する概念 |
| グループ | 表示上の便宜のために複数要素をまとめたもの。コンテナとは異なりレイアウト機能を持たない |
| レイアウトエンジン | コンテナ内の子要素の配置を自動計算するロジック。Flexbox的/Grid的な配置を担当 |
| DOMオーバーレイ | Canvas上にHTML DOM要素を重ねて表示する手法。テキスト編集やフォーム要素など、Canvas単体では扱いにくいUI要素の表示に使用する |
| Konva.js Group | Konva.jsが提供する描画レベルのノードグルーピング機能。本ツールの「グループ」と「コンテナ」両方の描画上の基盤として使用するが、両概念の区別はドメイン層のデータモデルで管理する |
| SPA | Single Page Application。1つのHTMLページ上でJavaScriptにより画面遷移を行うWebアプリケーション |
| PWA | Progressive Web App。Service Worker等の技術でオフライン動作やインストールを可能にするWebアプリケーション |
| ビューポートカリング | 画面外（ビューポート外）の要素の描画をスキップすることでパフォーマンスを向上させる手法 |
| スマートガイド | 要素の移動時に他の要素との位置関係（辺の揃い、等間隔等）を検出し、ガイドラインを表示して整列を補助する機能 |
| シェイプキャッシング | Konva.js の `cache()` メソッドを使用して、複雑な図形やグループをオフスクリーンキャンバスにキャッシュし、再描画コストを削減する手法 |
| ドリルダウン選択 | ネストされたコンテナ/グループ構造において、ダブルクリックで1段深い階層に入り、ESCで戻る段階的な選択操作方式 |
| 選択コンテキスト | ドリルダウン選択において、現在操作中の階層を示す状態。ヒットテストの対象範囲を制限する |
| トランザクション（Undo） | ドラッグやリサイズなど連続的に発生する変更を、Undo時に1回の操作として扱うためにまとめる機構 |
| プロパティ差分（Patch）方式 | Undo/Redoの実装方式。操作ごとにコマンドクラスを定義せず、変更前後のプロパティ差分を汎用的に記録するアプローチ |
| ExportNode | エクスポート変換パイプラインの中間表現。ElementNode（フラットマップ）をツリー構造に変換し、レイアウト計算結果（座標・サイズ）を付与したもの |
| 変換パイプライン | ElementNode → ExportNode（中間表現）→ ターゲット固有AST → 出力文字列/Blob の段階的変換処理 |
| XamlFrameworkAdapter | 4つのXAMLフレームワーク（WPF, WinUI 3, .NET MAUI, Avalonia UI）間の差異（名前空間、コントロール名、プロパティ名等）を吸収するアダプタ |
| ツールモード | 要素パレットから図形/UIコンポーネントを選択した際に入る配置モード。キャンバス上でクリック/ドラッグすると要素を配置し、ESCで通常の選択モードに戻る |

※ 各テーマの設計が進むにつれて、用語を追加する。
